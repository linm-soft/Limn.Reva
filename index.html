Câu Hỏi Thường GặpFrequently Asked Questions

Câu hỏi thường gặp

Frequently asked questions

REVA Platform có thay thế hệ thống POS hoặc ERP hiện tại không? Does REVA Platform replace the current POS or ERP? +
Không thay POS. Linm ERP là hệ thống kế toán và báo cáo chính thức; REVA Platform kết nối POS, membership và đồng bộ dữ liệu tự động. POS stays. Linm ERP is the official accounting and reporting system; REVA Platform connects POS, membership, and automates data sync.
Khách hàng có cần tạo tài khoản để đặt Bữa Cơm Nhà không? Do customers need to create an account to order Bữa Cơm Nhà? +
Không. Khách chỉ cần điền số điện thoại để đặt hàng lần đầu. Hệ thống tự tạo profile theo SĐT. Khách được gợi ý đăng ký Signature sau khi đặt hàng thành công lần đầu. No. Customers just need to enter their phone number to place their first order. The system automatically creates a profile from the phone number. Customers are suggested to register for Signature after their first successful order.
Bữa Cơm Nhà hết mùa thì hệ thống làm gì? What happens when the Bữa Cơm Nhà season ends? +
Admin tắt chương trình "FRESH_ORDER" trong Program Registry. Platform core (Membership, Order engine, Inventory) không thay đổi. Chương trình tiếp theo (Flash Sale, Pre-order...) được bật lên — không cần deploy code mới. Admin disables the "FRESH_ORDER" program in the Program Registry. The platform core (Membership, Order engine, Inventory) remains unchanged. The next program (Flash Sale, Pre-order...) is activated — no new code deployment needed.
Tiền trong ví Signature có an toàn không? Nếu hệ thống lỗi thì sao? Is money in the Signature wallet safe? What if the system crashes? +
Ví Signature dùng ledger append-only — mọi biến động được ghi vĩnh viễn, không ai có thể xóa hay sửa. Database không cho phép UPDATE/DELETE trên bảng ledger. Mọi thay đổi đều có audit log đầy đủ. Signature Wallet uses an append-only ledger — every change is permanently recorded, no one can delete or modify it. The database doesn't allow UPDATE/DELETE on the ledger table. Every change has a full audit log.
Đối tác có thể xem dữ liệu của nhau không? Can partners see each other's data? +
Không. Tài khoản đối tác được gắn với partner_id cụ thể. API tự động lọc dữ liệu theo partner_id — không cần filter thủ công, không thể xem sai dữ liệu kể cả khi URL bị đoán. No. Partner accounts are tied to a specific partner_id. The API automatically filters data by partner_id — no manual filtering needed, cannot see wrong data even if URLs are guessed.
Khi nào có thể chạy thực tế? When can this go live? +
30 ngày để có MVP Bữa Cơm Nhà tại 1 cửa hàng: trang đặt hàng, quản lý đơn nội bộ, mã đơn tự sinh, xuất báo cáo cuối ca. 60 ngày: Ví Signature và Gói Cơm Nhà+ đầy đủ. 90 ngày: Dashboard CEO/kế toán và dữ liệu CRM cơ bản. 30 days for Bữa Cơm Nhà MVP at 1 store: ordering page, internal order management, auto-generated order codes, end-of-shift reports. 60 days: Full Signature Wallet and Cơm Nhà+ plans. 90 days: CEO/accounting dashboard and basic CRM data.
Hạ tầng và chi phí vận hành? Infrastructure and operating costs? +
Phase 1 deploy trên Railway hoặc VPS: khoảng 500–700,000 VNĐ/tháng cho hạ tầng. Hệ thống được containerize bằng Docker — dễ dàng scale hoặc chuyển sang Kubernetes khi cần. Nhân sự: Tech lead + 1 Backend + 1 Frontend. Phase 1 deployed on Railway or VPS: approximately 500–700,000 VND/month for infrastructure. System is containerized with Docker — easy to scale or migrate to Kubernetes when needed. Team: Tech lead + 1 Backend + 1 Frontend.

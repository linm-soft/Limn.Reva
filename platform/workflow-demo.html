Event-Job System — Hàng Đợi PostgreSQL Tự Xây

Event-Job System — Custom PostgreSQL Queue

Không dùng EMQX · Không dùng RabbitMQ · Transactional Outbox + BackgroundService Worker

No EMQX · No RabbitMQ · Transactional Outbox + BackgroundService Worker

Publisher → Worker → Handler
1. Publisher (in service — same transaction)
dbContext.Orders.Add(order);
eventPublisher.PublishAsync(new OrderCreatedEvent{...}, dbContext);
await dbContext.SaveChangesAsync(); // atomic
2. EventJobWorker (BackgroundService — polls 500ms)
// Fetch PENDING batch
var jobs = await FetchPendingAsync(batchSize: 50);
// Parallel dispatch
await Parallel.ForEachAsync(jobs, maxDegree: 10,
  async (job, ct) => await ProcessAsync(job, ct));
3. Handler (idempotent)
if (await _processed.ExistsAsync(job.Id)) return;
await handler.HandleAsync(job, ct);
await _processed.MarkAsync(job.Id, ct);

Bảo Mật & Giới Hạn Tốc Độ

Security & Rate Limiting

3 lớp bảo vệ · JWT RS256 · CORS whitelist · Ledger chỉ ghi thêm

3 protection layers · JWT RS256 · CORS whitelist · Append-only ledger

Layer 1: reva-gateway (api.reva.com)

HTTPS + HSTS CORS: reva.com + event.reva.com only JWT validation (except /api/public/*) Rate limit per IP + per user Request size: max 1MB Inject X-User-Id, X-Store-Id headers
Rate Limiting
POST /api/public/orders (per IP)
3 / 5 per min
GET /api/public/* (per IP)
27 / 60 per min
Authenticated API (per user)
180 / 300 per min
POST /api/auth/* (brute-force protect)
9 / 60 per min

Layer 2: Business Services (reva-order, reva-membership...)

FluentValidation on all DTOs Permission check per operation EF Core parameterized queries Store isolation (EF global filter) App rate limit: 3 pending orders per phone/24h Phone format validation (Vietnamese) PII masking in logs (09***1234)
// FluentValidation — public order
RuleFor(x => x.Phone).Matches(@"^(0|\+84)[3-9]\d{8}$");
RuleFor(x => x.Items).Must(i => i.Count <= 20);
RuleFor(x => x.DeliverySlot).GreaterThan(DateTime.UtcNow.AddHours(1));

Layer 3: Data (PostgreSQL)

Append-only: wallet_ledgers (no UPDATE/DELETE) Append-only: event_jobs, transaction_audit_logs DB user: no DROP/TRUNCATE/ALTER SSL required for DB connections Secrets via env vars only (never appsettings.json) Audit trail: every mutation logged
-- App user: NO write on ledger tables
REVOKE UPDATE, DELETE ON wallet_ledgers FROM reva_app_user;
REVOKE UPDATE, DELETE ON transaction_audit_logs FROM reva_app_user;
